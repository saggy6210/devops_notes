## Design Components

### Section Cards
Each major section uses a card with header and body:
```html
<section class="section-card">
    <div class="section-header">
        <div class="section-icon"><i class="fas fa-icon"></i></div>
        <h2>Section Title</h2>
    </div>
    <div class="section-body">
        <!-- Content here -->
    </div>
</section>
```

### Info Boxes
```html
<div class="info-box">
    <i class="fas fa-info-circle"></i>
    <p>Important information here</p>
</div>
```

### Command Cards
```html
<div class="command-card">
    <code>command here</code>
    <span class="command-desc">Description</span>
</div>
```

### Q&A Cards
```html
<div class="qa-card">
    <div class="qa-question">
        <span class="qa-badge">Q1</span>
        <h4>Question text?</h4>
    </div>
    <div class="qa-answer">
        <p>Answer text</p>
    </div>
</div>
```

### Concept Grid
```html
<div class="concept-grid">
    <div class="concept-card">
        <div class="concept-icon"><i class="fas fa-icon"></i></div>
        <h4>Concept Name</h4>
        <p>Description</p>
    </div>
</div>
```

## Content Requirements

1. **Hero Section**: Day badge, title with gradient text, description, meta info (reading time, Q&A count, scenarios count)

2. **Core Concepts**: Clear explanation with info-box, analogy-card for simple understanding

3. **Key Points**: Use concept-grid with icons, or bullet lists with visual styling

4. **Commands Section**: Use command-card components with syntax highlighting

5. **Interview Q&A**: 5-7 questions using qa-card component with badges Q1, Q2, etc.

6. **Scenario Questions**: 2-3 real-world scenarios using scenario-card component

7. **Navigation**: Previous/Next day buttons at bottom

8. **Footer**: Standard footer with copyright "© 2026 Sagar Chavan. DevOps Learning Hub."
