AWS RDS & Databases

Managed relational databases on AWS — RDS, Aurora, and best practices.

15-20 minReading time
5+ Q&AInterview prep
2 ScenariosReal-world

RDS Overview

Amazon RDS is a managed relational database service supporting MySQL, PostgreSQL, MariaDB, Oracle, SQL Server, and Aurora.

💡

RDS is like having a DBA on staff 24/7 — backups, patching, failover are all handled for you.

Key Concepts

Multi-AZ

Synchronous standby in another AZ for HA, automatic failover

Read Replica

Asynchronous copies for read scaling, can be cross-region

Aurora

AWS-native engine, MySQL/Postgres compatible, 5x faster

DynamoDB

Managed NoSQL, single-digit ms latency, auto-scaling

ElastiCache

In-memory cache (Redis or Memcached) for hot data

Common Commands

aws rds create-db-instance --db-instance-identifier mydb --engine mysql --master-username admin --master-user-password xxx --allocated-storage 20 Create RDS
aws rds describe-db-instances List databases
aws rds create-db-snapshot --db-snapshot-identifier mysnap --db-instance-identifier mydb Snapshot
aws rds modify-db-instance --db-instance-identifier mydb --multi-az Enable Multi-AZ
aws dynamodb list-tables List DynamoDB tables

Interview Questions

1

Multi-AZ vs Read Replica?

Multi-AZ: standby in another AZ for HA — synchronous, no read traffic. Read Replica: async, used to offload reads, can promote to primary.
2

Aurora vs RDS?

Aurora is AWS's purpose-built engine — auto-scaling storage to 128TB, 6 copies across 3 AZs, faster failover (seconds vs minutes).
3

RDS vs DynamoDB?

RDS = relational (SQL, joins, transactions). DynamoDB = NoSQL key-value/document, infinite scale, no joins.
4

How do RDS backups work?

Automated daily backups (configurable retention 1–35 days) and manual snapshots. Backups are stored in S3.
5

How to migrate a DB to AWS?

Use AWS DMS (Database Migration Service) — supports homogeneous and heterogeneous migrations with continuous replication.

Real-World Scenarios

RDS instance running slow

🔍 Approach
  1. Check CloudWatch CPU and IOPS metrics
  2. Use Performance Insights to identify slow queries
  3. Add indexes on frequently queried columns
  4. Consider scaling instance class or adding read replicas

DB connection limit reached

🔍 Approach
  1. Increase max_connections parameter
  2. Use RDS Proxy for connection pooling
  3. Audit application connection leaks
  4. Scale to a larger instance class

Quick Tips

Enable encryption at rest and in transit

Test backup restores quarterly

Place RDS in private subnets only

Use RDS Proxy to manage connection storms
