Day 02: EC2 & S3

EC2 (Elastic Compute Cloud)

  • Virtual machines in AWS, called "instances".
  • Instance types, AMIs, key pairs, security groups.
  • Launching, stopping, and terminating instances.

S3 (Simple Storage Service)

  • Object storage for files, backups, and static websites.
  • Buckets, objects, versioning, lifecycle policies.
  • Access control and S3 policies.

Hands-on

  1. Launch an EC2 instance and connect via SSH.
  2. Create an S3 bucket and upload/download files.

Interview Tips

  • Know EC2 pricing models (On-Demand, Reserved, Spot).
  • Understand S3 storage classes and use cases.
