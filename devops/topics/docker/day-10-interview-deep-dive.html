Core Concepts Questions

1

What is Docker and why use it?

Docker is a containerization platform that packages applications with their dependencies into containers - lightweight, portable, isolated environments.

Why use Docker:

  • Consistency across environments (dev, staging, prod)
  • Fast startup (seconds vs minutes for VMs)
  • Resource efficiency (share host kernel)
  • Microservices architecture support
  • Easy CI/CD integration
2

Explain Docker architecture

Docker uses a client-server architecture:

  • Docker Client: CLI tool (docker command), sends commands via REST API
  • Docker Daemon (dockerd): Background service that manages containers, images, networks, volumes
  • Docker Registry: Stores images (Docker Hub, ECR, ACR)
  • containerd: Container runtime managing container lifecycle
  • runc: Low-level runtime that creates containers

4

What is the difference between CMD and ENTRYPOINT?

  • CMD: Default command/arguments, can be overridden completely
  • ENTRYPOINT: Fixed executable, CMD becomes arguments

Example:

ENTRYPOINT ["python"]
CMD ["app.py"]

Run docker run myapp test.py executes python test.py

Use ENTRYPOINT for the main executable, CMD for default arguments.
5

What are Docker layers?

Each instruction in a Dockerfile creates a layer:

  • Layers are read-only and cached
  • Containers add a writable layer on top
  • Unchanged layers are reused (faster builds)
  • Layer order matters - put changing content last

View layers: docker history myimage

Networking & Storage Questions

6

Explain Docker network types

  • bridge: Default. Isolated network, containers communicate via IP
  • host: Container uses host network directly (no isolation)
  • none: No networking
  • overlay: Multi-host networking (Swarm/K8s)
  • macvlan: Container gets its own MAC address
7

Difference between volumes and bind mounts?

  • Volumes: Managed by Docker, stored in /var/lib/docker/volumes/, portable, supports drivers
  • Bind mounts: Maps host directory directly, depends on host path, changes reflect immediately

Best practice: Volumes for production, bind mounts for development (hot reload).

8

How do containers communicate in Docker Compose?

Docker Compose creates a default network where:

  • Services can reach each other by service name as hostname
  • Example: http://db:5432 connects to database service
  • Built-in DNS resolution

Security Questions

9

How do you secure Docker containers?

  1. Run as non-root user (USER directive)
  2. Use minimal base images (Alpine, Distroless)
  3. Scan images for CVEs (Trivy, Snyk)
  4. Drop capabilities: --cap-drop=ALL
  5. Use read-only filesystem: --read-only
  6. Limit resources: --memory=256m
  7. Never use --privileged
  8. Keep images updated
10

Why is running containers as root dangerous?

  • Container escape gives root access to host
  • Can modify system files if volume mounted
  • Violates principle of least privilege
  • Many CVEs target root containers

Fix: Always add USER nonroot in Dockerfile.

11

What is Docker Content Trust?

DCT provides image signing and verification:

  • Enable: export DOCKER_CONTENT_TRUST=1
  • Only signed images can be pulled/pushed
  • Prevents man-in-the-middle attacks
  • Uses Notary for key management

Scenario-Based Questions

Container exits immediately - how to debug?

  1. Check logs: docker logs container_name
  2. Check exit code: docker inspect container | grep ExitCode
  3. Run interactively: docker run -it image /bin/sh
  4. Check if main process is foreground (not daemon)
  5. Verify CMD/ENTRYPOINT syntax

Docker disk space is full - what to do?

  1. Check usage: docker system df
  2. Remove stopped containers: docker container prune
  3. Remove unused images: docker image prune -a
  4. Remove unused volumes: docker volume prune
  5. Nuclear option: docker system prune -a --volumes
  6. Set log rotation in daemon.json

Container keeps getting OOMKilled

🔍 Diagnose
  1. Confirm OOM: docker inspect container | grep OOMKilled
  2. Monitor: docker stats container
✅ Solutions
  • Increase limit: --memory=512m
  • Add swap: --memory-swap=1g
  • Fix memory leak in application
  • Optimize application memory usage

How to reduce Docker image size?

  1. Use multi-stage builds
  2. Use Alpine or Distroless base images
  3. Combine RUN commands to reduce layers
  4. Clean up in same layer: rm -rf /var/cache/*
  5. Use .dockerignore
  6. Install only production dependencies
  7. Use --no-install-recommends for apt
