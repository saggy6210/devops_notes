Docker Logging & Debugging

Master container troubleshooting techniques. Learn log drivers, debugging strategies, and common issue resolution.

20-25 minReading time
5 Q&AInterview prep
DebuggingEssential skill

Container Logging Basics

Docker captures stdout and stderr from the container's main process. Logs are stored by the configured logging driver and can be viewed with docker logs.

docker logs <container> View container logs
docker logs -f <container> Follow logs in real-time
docker logs --tail 100 <container> Last 100 lines
docker logs --since 1h <container> Logs from last hour
docker logs -t <container> Include timestamps

Logging Drivers

Docker supports multiple logging drivers for different use cases:

json-file (default)

JSON formatted logs stored on disk

--log-driver=json-file

syslog

Send to syslog server

--log-driver=syslog

awslogs

AWS CloudWatch Logs

--log-driver=awslogs

fluentd

Forward to Fluentd/Fluent Bit

--log-driver=fluentd
Configure log rotation (json-file) 
# Run with log limits
docker run -d \
  --log-driver=json-file \
  --log-opt max-size=10m \
  --log-opt max-file=3 \
  nginx

# Or in daemon.json (global config)
{
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "10m",
    "max-file": "3"
  }
}

Debugging Commands

docker exec -it <container> /bin/sh Interactive shell into container
docker inspect <container> Detailed container info
docker stats Real-time resource usage
docker top <container> Running processes in container
docker diff <container> Changed files in container
docker events Real-time Docker events
Useful inspect queries 
# Get container IP
docker inspect -f '{{range.NetworkSettings.Networks}}{{.IPAddress}}{{end}}' container

# Get exit code
docker inspect -f '{{.State.ExitCode}}' container

# Get environment variables
docker inspect -f '{{.Config.Env}}' container

# Get mount points
docker inspect -f '{{json .Mounts}}' container | jq

Common Issues & Solutions

Container exits immediately

🔍 Debug Steps
  1. Check logs: docker logs container_name
  2. Check exit code: docker inspect -f '{{.State.ExitCode}}' container
  3. Run interactively: docker run -it myimage /bin/sh
⚠️ Common Causes
  • Main process exits (no foreground process)
  • Missing environment variables
  • Permission issues on volumes

Container killed (OOMKilled)

🔍 Debug Steps
  1. Check: docker inspect container | grep OOMKilled
  2. Monitor: docker stats container
✅ Solutions
  • Increase memory limit: --memory=512m
  • Fix memory leak in application
  • Use swap: --memory-swap=1g

Container can't reach network

🔍 Debug Steps
  1. Check network: docker network ls
  2. Inspect network: docker network inspect bridge
  3. Test DNS: docker exec container nslookup google.com
✅ Solutions
  • Check container is on correct network
  • Verify firewall rules on host
  • Check Docker daemon DNS config

Interview Questions

1

How do you troubleshoot a container that won't start?

  1. docker logs container - Check for errors
  2. docker inspect container - Check state and exit code
  3. docker run -it image /bin/sh - Run interactively
  4. Check if CMD/ENTRYPOINT is correct
  5. Verify environment variables and volumes
2

What logging drivers do you know?

  • json-file: Default, logs to JSON files
  • syslog: Sends to syslog daemon
  • journald: Linux journal
  • awslogs: AWS CloudWatch
  • fluentd: Fluentd collector
  • splunk: Splunk HTTP endpoint
3

How do you get a shell into a running container?

  • docker exec -it container /bin/bash - Bash shell
  • docker exec -it container /bin/sh - Shell (Alpine)
  • docker attach container - Attach to main process (caution!)
Use exec, not attach! Exiting attach can stop the container.
4

How do you monitor container resource usage?

  • docker stats - Live resource usage
  • docker stats --no-stream - Snapshot
  • cAdvisor - Container metrics collector
  • Prometheus + Grafana - Monitoring stack
5

What does OOMKilled mean?

OOMKilled = Out of Memory Killed

  • Container exceeded its memory limit
  • Linux kernel killed the process
  • Check with: docker inspect container | grep OOM

Solutions: Increase memory limit or fix application memory leak.

Quick Tips

Always set log rotation: --log-opt max-size=10m

Debug with: docker run --rm -it image sh

Use docker stats before setting resource limits

Log to stdout in containers, not files
