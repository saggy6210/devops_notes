Dockerfile Fundamentals

A Dockerfile is a text document containing instructions to build a Docker image. Each instruction creates a layer in the image.

Essential Instructions

FROM

Base image for the build

FROM node:18-alpine

WORKDIR

Set working directory

WORKDIR /app

COPY

Copy files from host

COPY package*.json ./

RUN

Execute commands

RUN npm install

EXPOSE

Document port

EXPOSE 3000

CMD

Default run command

CMD ["node", "app.js"]

Layer Caching & Optimization

Docker caches each layer. If a layer changes, all subsequent layers are rebuilt. Order matters!

Bad - Rebuilds everything on code change Slow 
FROM node:18
WORKDIR /app
COPY . .  # Code changes = npm install again
RUN npm install
CMD ["node", "app.js"]
Good - Cache dependencies Fast 
FROM node:18
WORKDIR /app
COPY package*.json ./  # Deps first
RUN npm install         # Cached!
COPY . .               # Code last
CMD ["node", "app.js"]

Combine RUN Commands

Bad - Multiple layers Large 
RUN apt-get update
RUN apt-get install -y curl
RUN apt-get install -y git
RUN rm -rf /var/lib/apt/lists/*
Good - Single layer Small 
RUN apt-get update && \
    apt-get install -y \
        curl \
        git && \
    rm -rf /var/lib/apt/lists/*

Choosing Base Images

Image Size Comparison 
node:18          → ~950MB  # Full Debian + Node
node:18-slim     → ~200MB  # Minimal Debian
node:18-alpine   → ~120MB  # Alpine Linux (smallest)
gcr.io/distroless/nodejs18 → ~110MB # No shell!
Use Alpine images for smaller size. Use Distroless for maximum security (no shell = harder to exploit).

Security Best Practices

Run as Non-Root User

Dockerfile with non-root user Secure 
FROM node:18-alpine

# Create non-root user
RUN addgroup -S appgroup && adduser -S appuser -G appgroup

WORKDIR /app
COPY --chown=appuser:appgroup . .
RUN npm ci --only=production

# Switch to non-root user
USER appuser

EXPOSE 3000
CMD ["node", "app.js"]

Interview Questions

1

What's the difference between CMD and ENTRYPOINT?

  • CMD: Default command, can be overridden at runtime
  • ENTRYPOINT: Fixed executable, CMD becomes arguments

Example with both:

ENTRYPOINT ["python"]
CMD ["app.py"]

Running docker run myapp test.py executes python test.py

2

What's the difference between COPY and ADD?

  • COPY: Simple copy from host to container
  • ADD: COPY + auto-extract archives + fetch URLs
Best practice: Always use COPY unless you specifically need ADD's features.
3

How does Docker layer caching work?

  1. Docker builds images layer by layer
  2. Each instruction creates a new layer
  3. Layers are cached and reused if unchanged
  4. If a layer changes, all subsequent layers are rebuilt
Put frequently changing instructions (COPY code) at the end of Dockerfile.
4

What is the purpose of ARG and ENV?

  • ARG: Build-time variables, not available in running container
  • ENV: Runtime environment variables, persist in container

Example:

ARG VERSION=1.0
ENV APP_VERSION=$VERSION

Build with: docker build --build-arg VERSION=2.0 .

5

How do you reduce Docker image size?

  1. Use smaller base images (Alpine, Distroless)
  2. Combine RUN commands to reduce layers
  3. Use multi-stage builds (covered in Day 5)
  4. Clean up cache in same layer: rm -rf /var/lib/apt/lists/*
  5. Use .dockerignore to exclude unnecessary files
  6. Install only production dependencies: npm ci --only=production

Quick Tips

Always pin versions: FROM node:18.17.0-alpine

Use npm ci not npm install for reproducible builds

Add health checks: HEALTHCHECK CMD curl -f http://localhost/ || exit 1

Label images: LABEL maintainer="team@company.com"
