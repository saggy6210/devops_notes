What is Docker Compose?

Docker Compose is a tool for defining and running multi-container applications. You define services, networks, and volumes in a YAML file and manage them with single commands.

Compose File Structure

docker-compose.yml - Full Stack App 
# Docker Compose file version
version: "3.9"

services:
  # Node.js Application
  web:
    build: .
    ports:
      - "3000:3000"
    environment:
      - DATABASE_URL=postgres://user:pass@db:5432/app
      - REDIS_URL=redis://cache:6379
    depends_on:
      - db
      - cache
    volumes:
      - .:/app
      - /app/node_modules

  # PostgreSQL Database
  db:
    image: postgres:15-alpine
    environment:
      - POSTGRES_USER=user
      - POSTGRES_PASSWORD=pass
      - POSTGRES_DB=app
    volumes:
      - pgdata:/var/lib/postgresql/data
    ports:
      - "5432:5432"

  # Redis Cache
  cache:
    image: redis:7-alpine
    ports:
      - "6379:6379"

volumes:
  pgdata:

Essential Commands

docker compose up -d Start all services in background
docker compose down Stop and remove containers
docker compose ps List running services
docker compose logs -f web Follow logs of a service
docker compose exec web sh Shell into running service
docker compose build --no-cache Rebuild images without cache
docker compose down -v Stop and remove with volumes
docker compose pull Pull latest images

Profiles & Overrides

Environment-Specific Configuration

docker-compose.override.yml (Development) 
services:
  web:
    build:
      context: .
      target: development
    volumes:
      - .:/app  # Hot reload
    environment:
      - DEBUG=true

  db:
    ports:
      - "5432:5432"  # Expose for local tools

Using Profiles

docker-compose.yml with profiles 
services:
  web:
    image: myapp
    
  debug:
    image: busybox
    profiles:
      - debug
    
  tests:
    image: myapp:test
    profiles:
      - test

Run with profile: docker compose --profile debug up

Interview Questions

1

What is the difference between docker-compose up and docker-compose run?

  • up: Starts all services defined in compose file
  • run: Runs a one-off command on a service (like running tests)

Example: docker compose run web npm test

2

How does depends_on work?

depends_on controls startup order, but NOT readiness:

  • Services start in dependency order
  • Doesn't wait for service to be "ready" (e.g., DB accepting connections)
Use healthchecks with condition: service_healthy for proper dependency handling.
3

How do services communicate in Docker Compose?

Docker Compose creates a default network where:

  • Services can reach each other by service name
  • Example: db:5432 connects to database service
  • No need for localhost or IP addresses

Custom networks: services only see others on same network

4

What's the difference between ports and expose?

  • ports: Publishes port to host machine (external access)
  • expose: Makes port available only to linked services (internal only)
Security tip: Don't expose database ports to host in production!
5

How do you handle secrets in Docker Compose?

Options for managing secrets:

  1. .env file: docker compose --env-file .env.prod up
  2. Docker secrets: Available in Swarm mode
  3. External secrets: HashiCorp Vault, AWS Secrets Manager

Never commit secrets to version control!

Quick Tips

Use restart: unless-stopped for auto-restart

Use .env file for environment variables

Add healthchecks for proper dependency ordering

Always pin image versions in production
