What is Helm?

Helm is the package manager for Kubernetes. It helps you define, install, and upgrade Kubernetes applications using "charts" - packages of pre-configured K8s resources.

Key Concepts

  • Chart - A package containing K8s resource definitions
  • Release - An instance of a chart running in a cluster
  • Repository - A place where charts are stored and shared
  • Values - Configuration for customizing chart deployments

Essential Helm Commands

Repository Management # Add repository helm repo add bitnami https://charts.bitnami.com/bitnami helm repo add stable https://charts.helm.sh/stable # Update repos helm repo update # List repos helm repo list # Search charts helm search repo nginx helm search hub prometheus # Search Artifact Hub
Install & Manage Releases # Install a chart helm install my-nginx bitnami/nginx helm install my-nginx bitnami/nginx -n web --create-namespace # Install with custom values helm install my-app ./mychart -f values.yaml helm install my-app ./mychart --set replicaCount=3,image.tag=v2 # Upgrade release helm upgrade my-nginx bitnami/nginx --set replicaCount=5 helm upgrade --install my-nginx bitnami/nginx # Install if not exists # Rollback helm rollback my-nginx 1 helm rollback my-nginx # Previous version
Release Management # List releases helm list helm list -A # All namespaces helm list --pending # Pending releases # Get release info helm status my-nginx helm history my-nginx helm get values my-nginx helm get manifest my-nginx # Uninstall helm uninstall my-nginx helm uninstall my-nginx --keep-history

Chart Structure

mychart/ ├── Chart.yaml # Chart metadata ├── values.yaml # Default configuration values ├── charts/ # Chart dependencies ├── templates/ # K8s manifest templates │ ├── deployment.yaml │ ├── service.yaml │ ├── ingress.yaml │ ├── _helpers.tpl # Template helpers │ ├── NOTES.txt # Post-install notes │ └── tests/ # Test files ├── .helmignore # Files to ignore when packaging └── README.md # Documentation
Chart.yaml apiVersion: v2 name: myapp description: A Helm chart for my application type: application version: 0.1.0 # Chart version appVersion: "1.0.0" # Application version dependencies: - name: postgresql version: "12.x.x" repository: https://charts.bitnami.com/bitnami condition: postgresql.enabled

Helm Templating

values.yaml replicaCount: 3 image: repository: nginx tag: "1.21" pullPolicy: IfNotPresent service: type: ClusterIP port: 80 resources: limits: cpu: 100m memory: 128Mi requests: cpu: 50m memory: 64Mi
templates/deployment.yaml apiVersion: apps/v1 kind: Deployment metadata: name: {{ include "myapp.fullname" . }} labels: {{- include "myapp.labels" . | nindent 4 }} spec: replicas: {{ .Values.replicaCount }} selector: matchLabels: {{- include "myapp.selectorLabels" . | nindent 6 }} template: metadata: labels: {{- include "myapp.selectorLabels" . | nindent 8 }} spec: containers: - name: {{ .Chart.Name }} image: "{{ .Values.image.repository }}:{{ .Values.image.tag }}" imagePullPolicy: {{ .Values.image.pullPolicy }} ports: - containerPort: 80 resources: {{- toYaml .Values.resources | nindent 12 }}

Template Functions

  • {{ .Values.key }} - Access values
  • {{ .Release.Name }} - Release name
  • {{ .Chart.Name }} - Chart name
  • {{ include "template" . }} - Include named template
  • {{ toYaml .Values.x | nindent 4 }} - Convert to YAML
  • {{- if .Values.enabled }} - Conditionals
  • {{- range .Values.items }} - Loops

Chart Development

Always use helm template or --dry-run to preview generated manifests before applying to the cluster.

Interview Q&A

Q: What is the difference between helm install and helm upgrade?

helm install: Creates a new release. Fails if release name already exists.

helm upgrade: Updates an existing release. Fails if release doesn't exist.

helm upgrade --install: Combines both - installs if not exists, upgrades if it does. Ideal for CI/CD.

Q: How do you rollback a failed Helm release?
  1. helm history my-release - View revision history
  2. helm rollback my-release <revision> - Rollback to specific revision
  3. helm rollback my-release - Rollback to previous version

Helm keeps history of releases for rollback purposes.

Q: What are Helm hooks?

Hooks allow you to intervene at certain points in a release lifecycle:

  • pre-install - Before templates are rendered
  • post-install - After all resources are loaded
  • pre-upgrade/post-upgrade
  • pre-rollback/post-rollback
  • pre-delete/post-delete

Use annotation: helm.sh/hook: pre-install

Q: How do you pass values to a Helm chart?

Multiple methods, in order of precedence (last wins):

  1. Default values.yaml in chart
  2. -f values.yaml - Custom values file
  3. --set key=value - Command line
  4. --set-file key=filepath - Set from file
  5. --set-string key=value - Force string type
Q: What are some Helm alternatives?
  • Kustomize - Built into kubectl, template-free customization
  • Jsonnet - Data templating language
  • cdk8s - Define K8s resources using code (TypeScript, Python)
  • Pulumi - Infrastructure as code with real programming languages
  • Carvel (ytt) - YAML templating and overlays
