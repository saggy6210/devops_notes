🏗️ Introduction to Terraform

Learn the fundamentals of Infrastructure as Code with Terraform. Understand what Terraform is, its architecture, and how to get started with your first configuration.

What is Terraform?

Terraform is an open-source Infrastructure as Code (IaC) tool developed by HashiCorp. It allows you to define, provision, and manage infrastructure using a declarative configuration language called HCL (HashiCorp Configuration Language).

Declarative

Define the desired end state of your infrastructure, and Terraform figures out how to achieve it.

Cloud Agnostic

Works with AWS, Azure, GCP, and 100+ other providers using a consistent workflow.

Stateful

Maintains state to track resources and detect configuration drift.

Dependency Graph

Automatically handles resource dependencies and parallelizes operations.

Installing Terraform

Ubuntu/Debian

Install via APT 
# Add HashiCorp GPG key
wget -O- https://apt.releases.hashicorp.com/gpg | sudo gpg --dearmor -o /usr/share/keyrings/hashicorp-archive-keyring.gpg

# Add repository
echo "deb [signed-by=/usr/share/keyrings/hashicorp-archive-keyring.gpg] https://apt.releases.hashicorp.com $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/hashicorp.list

# Update and install
sudo apt update && sudo apt install terraform

macOS (Homebrew)

Install via Homebrew 
brew tap hashicorp/tap
brew install hashicorp/tap/terraform

Windows (Chocolatey)

Install via Chocolatey 
choco install terraform

Verify Installation

Check version 
terraform version

# Output:
# Terraform v1.6.0
# on linux_amd64

Core Terraform Workflow

1. Write

Define infrastructure in .tf files using HCL syntax.

2. Init

Initialize working directory and download providers.

3. Plan

Preview changes that will be made to infrastructure.

4. Apply

Execute the plan to create/modify infrastructure.

Basic Workflow Commands 
# Initialize Terraform (downloads providers)
terraform init

# Format configuration files
terraform fmt

# Validate configuration syntax
terraform validate

# Preview changes
terraform plan

# Apply changes
terraform apply

# Destroy infrastructure
terraform destroy

HCL Syntax Basics

HashiCorp Configuration Language (HCL) is Terraform's configuration language. It's designed to be human-readable and machine-friendly.

Block Structure

HCL Block Syntax 
# Block type "label1" "label2" {
#   argument = value
# }

resource "aws_instance" "web" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"
  
  tags = {
    Name = "HelloWorld"
  }
}

Common Block Types

Block Purpose
terraform Configure Terraform behavior and required providers
provider Configure cloud provider settings
resource Define infrastructure resources
variable Declare input variables
output Expose values to parent modules
data Query external data sources
module Import reusable modules
locals Define local values

Your First Terraform Configuration

Let's create a simple configuration that provisions a local file (no cloud account needed).

main.tf 
# Configure Terraform settings
terraform {
  required_version = ">= 1.0.0"
  required_providers {
    local = {
      source  = "hashicorp/local"
      version = "~> 2.0"
    }
  }
}

# Create a local file
resource "local_file" "hello" {
  filename = "${path.module}/hello.txt"
  content  = "Hello, Terraform!"
}

# Output the file path
output "file_path" {
  value       = local_file.hello.filename
  description = "The path to the generated file"
}
Run Your First Configuration 
# Initialize the directory
terraform init

# Preview the changes
terraform plan

# Apply the configuration
terraform apply

# Type 'yes' when prompted

# Verify the file was created
cat hello.txt

# Clean up
terraform destroy

Standard File Structure

Recommended Project Structure 
project/
├── main.tf          # Main configuration
├── variables.tf     # Variable definitions
├── outputs.tf       # Output definitions
├── providers.tf     # Provider configurations
├── versions.tf      # Version constraints
├── terraform.tfvars # Variable values (do not commit secrets!)
├── .gitignore       # Ignore state files, etc.
└── modules/         # Reusable modules
    └── vpc/
        ├── main.tf
        ├── variables.tf
        └── outputs.tf

Important Files

File Purpose Git?
*.tf Terraform configuration files ✓ Commit
terraform.tfstate Local state file ✗ Ignore
.terraform/ Downloaded providers ✗ Ignore
*.tfvars Variable values ⚠️ Depends (no secrets)
.terraform.lock.hcl Provider version lock ✓ Commit

Interview Q&A

Q1: What is Terraform and why is it used?

Terraform is an open-source Infrastructure as Code (IaC) tool by HashiCorp that allows you to define and provision infrastructure using declarative configuration files.

  • Enables version control for infrastructure
  • Provides consistent, reproducible deployments
  • Supports multiple cloud providers (AWS, Azure, GCP)
  • Automates infrastructure provisioning
  • Reduces manual errors and configuration drift
Q2: What is the difference between declarative and imperative IaC?

Declarative (Terraform): You define the desired end state, and the tool figures out how to achieve it.

Imperative (Scripts): You define step-by-step instructions to reach the desired state.

  • Declarative: "I want 3 servers" - Terraform adds/removes as needed
  • Imperative: "Create server 1, create server 2, create server 3"
  • Declarative is idempotent - running it multiple times produces the same result
Q3: Explain the Terraform workflow.

The core Terraform workflow consists of:

  • Write: Define infrastructure in .tf files
  • Init: Initialize directory, download providers
  • Plan: Preview changes without making them
  • Apply: Execute the plan to create resources
  • Destroy: Remove all managed infrastructure
Q4: What is the purpose of terraform init?

terraform init initializes a working directory containing Terraform configuration files. It:

  • Downloads required provider plugins
  • Downloads modules referenced in configuration
  • Initializes the backend for state storage
  • Creates .terraform directory for plugins
  • Creates .terraform.lock.hcl for version locking
Q5: What is Terraform state?

Terraform state is a file (terraform.tfstate) that maps your configuration to real-world resources.

  • Tracks resource metadata and IDs
  • Enables Terraform to detect drift
  • Determines what changes need to be applied
  • Should be stored remotely for team collaboration
  • Contains sensitive data - must be secured
