Variables & Outputs

📦 Variables, Outputs & Locals

Learn how to parameterize your Terraform configurations using input variables, expose values with outputs, and simplify code with local values.

Input Variables

Input variables make your Terraform configuration flexible and reusable. They allow you to customize deployments without modifying the code.

Basic Variable Definition

variables.tf 
# Simple variable
variable "region" {
  description = "AWS region for resources"
  type        = string
  default     = "us-east-1"
}

# Variable without default (required)
variable "instance_type" {
  description = "EC2 instance type"
  type        = string
}

# Variable with validation
variable "environment" {
  description = "Deployment environment"
  type        = string
  default     = "dev"
  
  validation {
    condition     = contains(["dev", "staging", "prod"], var.environment)
    error_message = "Environment must be dev, staging, or prod."
  }
}

# Sensitive variable
variable "db_password" {
  description = "Database password"
  type        = string
  sensitive   = true
}

Variable Types

Type Description Example
string Text value "us-east-1"
number Numeric value 3
bool True or false true
list(type) Ordered collection ["a", "b", "c"]
set(type) Unique collection ["a", "b", "c"]
map(type) Key-value pairs {key = "value"}
object({...}) Structured object {name = string, age = number}
tuple([...]) Fixed-length list [string, number, bool]
Complex Variable Types 
# List of strings
variable "availability_zones" {
  type    = list(string)
  default = ["us-east-1a", "us-east-1b", "us-east-1c"]
}

# Map of strings
variable "instance_tags" {
  type = map(string)
  default = {
    Environment = "dev"
    Project     = "myapp"
  }
}

# Object with specific structure
variable "instance_config" {
  type = object({
    instance_type = string
    ami_id        = string
    disk_size     = number
    public        = bool
  })
  default = {
    instance_type = "t3.micro"
    ami_id        = "ami-12345678"
    disk_size     = 20
    public        = true
  }
}

# List of objects
variable "users" {
  type = list(object({
    name  = string
    email = string
    admin = bool
  }))
  default = [
    { name = "john", email = "john@example.com", admin = true },
    { name = "jane", email = "jane@example.com", admin = false }
  ]
}

Setting Variable Values

Methods (in precedence order, highest to lowest)

-var Flag

terraform apply -var="region=us-west-2"

-var-file

terraform apply -var-file="prod.tfvars"

Environment

TF_VAR_region=us-west-2

Auto .tfvars

terraform.tfvars or *.auto.tfvars

Variable Files

terraform.tfvars 
# terraform.tfvars (auto-loaded)
region        = "us-west-2"
instance_type = "t3.small"
environment   = "prod"

availability_zones = [
  "us-west-2a",
  "us-west-2b"
]

instance_tags = {
  Project     = "webapp"
  Owner       = "devops"
  Environment = "production"
}

Environment Specific

prod.tfvars 
# Use: terraform apply -var-file="prod.tfvars"
environment   = "prod"

enable_monitoring = true

Using Variables in Resources

main.tf 
provider "aws" {
  region = var.region
}

resource "aws_instance" "web" {
  count         = var.instance_count
  ami           = var.ami_id
  instance_type = var.instance_type
  
  # Access list elements
  availability_zone = var.availability_zones[count.index % length(var.availability_zones)]
  
  # Merge maps
  tags = merge(var.instance_tags, {
    Name = "web-${count.index + 1}"
  })
}

resource "aws_security_group" "web" {
  name   = "${var.environment}-web-sg"
  vpc_id = var.vpc_id
  
  # Conditional ingress
  dynamic "ingress" {
    for_each = var.environment == "prod" ? [443] : [80, 443]
    content {
      from_port   = ingress.value
      to_port     = ingress.value
      protocol    = "tcp"
      cidr_blocks = var.allowed_cidrs
    }
  }
}

Output Values

Outputs expose values from your configuration to the command line, other configurations, or Terraform Cloud.

outputs.tf 
# Simple output
output "instance_id" {
  description = "ID of the EC2 instance"
  value       = aws_instance.web.id
}

# Output with sensitive value
output "db_password" {
  description = "Database password"
  value       = random_password.db.result
  sensitive   = true
}

# Multiple values
output "instance_ips" {
  description = "Public IPs of all instances"
  value       = aws_instance.web[*].public_ip
}

# Formatted output
output "connection_string" {
  description = "Database connection string"
  value       = "postgresql://${var.db_user}:${random_password.db.result}@${aws_db_instance.main.endpoint}/${var.db_name}"
  sensitive   = true
}

# Conditional output
output "load_balancer_dns" {
  description = "DNS of load balancer (if created)"
  value       = var.create_lb ? aws_lb.main[0].dns_name : null
}

# Map output
output "instance_details" {
  description = "Map of instance details"
  value = {
    for instance in aws_instance.web :
    instance.id => {
      public_ip  = instance.public_ip
      private_ip = instance.private_ip
      az         = instance.availability_zone
    }
  }
}

Accessing Outputs

CLI Commands 
# Show all outputs
terraform output

# Show specific output
terraform output instance_id

# Get raw value (for scripts)
terraform output -raw instance_id

# Get JSON format
terraform output -json

Local Values

Locals are computed values that can simplify your configuration by avoiding repetition and improving readability.

locals.tf 
locals {
  # Simple local
  project_name = "myapp"
  
  # Computed local
  name_prefix = "${var.environment}-${local.project_name}"
  
  # Common tags (merge with resource-specific tags)
  common_tags = {
    Environment = var.environment
    Project     = local.project_name
    ManagedBy   = "terraform"
    Owner       = var.owner
  }
  
  # Conditional logic
  instance_type = var.environment == "prod" ? "t3.large" : "t3.micro"
  
  # Complex computation
  subnet_ids = [
    for az in var.availability_zones :
    aws_subnet.private[az].id
  ]
  
  # Flattened list
  all_cidrs = flatten([
    var.vpc_cidrs,
    var.additional_cidrs
  ])
}

# Using locals in resources
resource "aws_instance" "web" {
  instance_type = local.instance_type
  
  tags = merge(local.common_tags, {
    Name = "${local.name_prefix}-web"
    Role = "webserver"
  })
}

resource "aws_s3_bucket" "data" {
  bucket = "${local.name_prefix}-data-bucket"
  
  tags = local.common_tags
}

Variable Validation

Validation Examples 
# Validate allowed values
variable "environment" {
  type = string
  validation {
    condition     = contains(["dev", "staging", "prod"], var.environment)
    error_message = "Environment must be dev, staging, or prod."
  }
}

# Validate string pattern
variable "bucket_name" {
  type = string
  validation {
    condition     = can(regex("^[a-z0-9][a-z0-9-]*[a-z0-9]$", var.bucket_name))
    error_message = "Bucket name must be lowercase alphanumeric with hyphens."
  }
}

# Validate number range
variable "instance_count" {
  type = number
  validation {
    condition     = var.instance_count >= 1 && var.instance_count <= 10
    error_message = "Instance count must be between 1 and 10."
  }
}

# Validate CIDR format
variable "vpc_cidr" {
  type = string
  validation {
    condition     = can(cidrnetmask(var.vpc_cidr))
    error_message = "VPC CIDR must be a valid CIDR notation."
  }
}

# Multiple validations
variable "ami_id" {
  type = string
  validation {
    condition     = length(var.ami_id) > 4 && substr(var.ami_id, 0, 4) == "ami-"
    error_message = "AMI ID must start with 'ami-'."
  }
}

Interview Q&A

Q1: What are the ways to set variable values in Terraform?

Variable values can be set in multiple ways (in order of precedence):

  • -var flag: terraform apply -var="region=us-west-2"
  • -var-file: terraform apply -var-file="prod.tfvars"
  • Environment variables: TF_VAR_region=us-west-2
  • terraform.tfvars: Auto-loaded from working directory
  • *.auto.tfvars: Auto-loaded alphabetically
  • Default value: Specified in variable definition
Q2: What is the difference between variables and locals?

Variables (var.name):

  • Input values provided by users
  • Can be set via CLI, tfvars, environment
  • Used for parameterization

Locals (local.name):

  • Computed values within configuration
  • Cannot be set externally
  • Used to avoid repetition and simplify logic
Q3: How do you mark a variable as sensitive?

Use the sensitive = true argument:

  • Prevents value from appearing in CLI output
  • Marks dependent outputs as sensitive
  • Still stored in state file (unencrypted)
  • Use remote backend with encryption for production
Q4: What is variable validation in Terraform?

Variable validation allows custom rules to ensure input values meet requirements:

  • Defined using validation blocks
  • condition: Expression that must be true
  • error_message: Message when validation fails
  • Validated during plan/apply phase
  • Can use functions like contains(), regex(), can()
Q5: How do you use outputs from one configuration in another?

Use terraform_remote_state data source:

  • Reference outputs from another state file
  • Configure backend details for remote state
  • Access via data.terraform_remote_state.name.outputs.value
  • Alternative: Use modules to share values directly
