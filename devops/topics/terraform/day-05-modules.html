Modules

📦 Terraform Modules

Learn to create reusable, maintainable infrastructure components with Terraform modules. Master module composition, versioning, and the Terraform Registry.

What are Modules?

A module is a container for multiple resources that are used together. Modules enable code reuse, organization, and encapsulation.

Reusability

Write once, use many times across projects and environments

Encapsulation

Hide complexity behind a simple interface

Organization

Organize code into logical components

Versioning

Track and control changes with version constraints

Module Structure

Standard Module Structure 
modules/
└── vpc/
    ├── main.tf           # Main resources
    ├── variables.tf      # Input variables
    ├── outputs.tf        # Output values
    ├── versions.tf       # Provider requirements

Creating a Module

VPC Module Example

modules/vpc/variables.tf 
variable "vpc_cidr" {
  description = "CIDR block for the VPC"
  type        = string
  default     = "10.0.0.0/16"
}

variable "environment" {
  description = "Environment name"
  type        = string
}

variable "public_subnet_cidrs" {
  description = "CIDR blocks for public subnets"
  type        = list(string)
  default     = ["10.0.1.0/24", "10.0.2.0/24"]
}

variable "private_subnet_cidrs" {
  description = "CIDR blocks for private subnets"
  type        = list(string)
  default     = ["10.0.10.0/24", "10.0.11.0/24"]
}

variable "enable_nat_gateway" {
  description = "Enable NAT Gateway for private subnets"
  type        = bool
  default     = true
}
modules/vpc/main.tf 
data "aws_availability_zones" "available" {
  state = "available"
}

locals {
  azs = slice(data.aws_availability_zones.available.names, 0, 2)
}

resource "aws_vpc" "main" {
  cidr_block           = var.vpc_cidr
  enable_dns_hostnames = true
  enable_dns_support   = true

  tags = {
    Name        = "${var.environment}-vpc"
    Environment = var.environment
  }
}

resource "aws_internet_gateway" "main" {
  vpc_id = aws_vpc.main.id

  tags = {
    Name = "${var.environment}-igw"
  }
}

resource "aws_subnet" "public" {
  count                   = length(var.public_subnet_cidrs)
  vpc_id                  = aws_vpc.main.id
  cidr_block              = var.public_subnet_cidrs[count.index]
  availability_zone       = local.azs[count.index]
  map_public_ip_on_launch = true

  tags = {
    Name = "${var.environment}-public-${count.index + 1}"
    Type = "public"
  }
}

resource "aws_subnet" "private" {
  count             = length(var.private_subnet_cidrs)
  vpc_id            = aws_vpc.main.id
  cidr_block        = var.private_subnet_cidrs[count.index]
  availability_zone = local.azs[count.index]

  tags = {
    Name = "${var.environment}-private-${count.index + 1}"
    Type = "private"
  }
}

resource "aws_nat_gateway" "main" {
  count         = var.enable_nat_gateway ? 1 : 0
  allocation_id = aws_eip.nat[0].id
  subnet_id     = aws_subnet.public[0].id

  tags = {
    Name = "${var.environment}-nat"
  }
}

resource "aws_eip" "nat" {
  count  = var.enable_nat_gateway ? 1 : 0
  domain = "vpc"
}
modules/vpc/outputs.tf 
output "vpc_id" {
  description = "ID of the VPC"
  value       = aws_vpc.main.id
}

output "vpc_cidr" {
  description = "CIDR block of the VPC"
  value       = aws_vpc.main.cidr_block
}

output "public_subnet_ids" {
  description = "IDs of public subnets"
  value       = aws_subnet.public[*].id
}

output "private_subnet_ids" {
  description = "IDs of private subnets"
  value       = aws_subnet.private[*].id
}

output "nat_gateway_ip" {
  description = "Public IP of NAT Gateway"
  value       = var.enable_nat_gateway ? aws_eip.nat[0].public_ip : null
}

Using Modules

Local Module

main.tf 
module "vpc" {
  source = "./modules/vpc"

  environment          = "prod"
  vpc_cidr             = "10.0.0.0/16"
  public_subnet_cidrs  = ["10.0.1.0/24", "10.0.2.0/24"]
  private_subnet_cidrs = ["10.0.10.0/24", "10.0.11.0/24"]
  enable_nat_gateway   = true
}

# Use module outputs
resource "aws_instance" "web" {
  ami           = "ami-12345678"
  instance_type = "t3.micro"
  subnet_id     = module.vpc.public_subnet_ids[0]
}

output "vpc_id" {
  value = module.vpc.vpc_id
}

    Environment = "dev"
  }
}

Module Composition

Composing Modules 
# Root module composes multiple child modules
module "networking" {
  source = "./modules/vpc"
  
  environment = var.environment
  vpc_cidr    = var.vpc_cidr
}

module "database" {
  source = "./modules/rds"
  
  environment   = var.environment
  vpc_id        = module.networking.vpc_id
  subnet_ids    = module.networking.private_subnet_ids
  instance_type = "db.t3.medium"
}

module "application" {
  source = "./modules/ecs"
  
  environment   = var.environment
  vpc_id        = module.networking.vpc_id
  subnet_ids    = module.networking.private_subnet_ids
  db_endpoint   = module.database.endpoint
}

Interview Q&A

Q1: What is a Terraform module and why use it?

A module is a container for multiple resources used together:

  • Promotes code reusability
  • Encapsulates complexity
  • Enables standardization
  • Simplifies maintenance
  • Root module = your working directory
Q2: What are the different module sources?
  • Local: ./modules/vpc
  • Registry: hashicorp/consul/aws
  • GitHub: github.com/org/repo
  • Git: git::https://example.com/repo.git
  • S3/GCS: Cloud storage buckets
  • HTTP: Direct URL to zip file
Q3: How do you version modules?

Version constraints ensure stability:

  • Registry: version = "~> 5.0"
  • Git: ?ref=v1.2.0 (tag/branch/commit)
  • Always pin versions in production
  • Use semantic versioning
Q4: How do you pass data between modules?
  • Module outputs expose values
  • Access via module.name.output_name
  • Creates implicit dependencies
  • Parent passes values to child via input variables
Q5: What is the Terraform Registry?

The Terraform Registry (registry.terraform.io):

  • Public repository of modules and providers
  • Verified modules from HashiCorp partners
  • Community-contributed modules
  • Documentation and examples included
  • Version history and changelogs
