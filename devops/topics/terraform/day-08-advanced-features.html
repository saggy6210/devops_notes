count vs for_each

count

Using count 
variable "instance_count" {
  default = 3
}

resource "aws_instance" "web" {
  count         = var.instance_count
  ami           = var.ami_id
  instance_type = "t3.micro"

  tags = {
    Name = "web-${count.index + 1}"
  }
}

# Access: aws_instance.web[0], aws_instance.web[1], etc.

for_each

Using for_each 
variable "instances" {
  type = map(object({
    instance_type = string
    ami           = string
  }))
  default = {
    web = { instance_type = "t3.micro", ami = "ami-123" }
    app = { instance_type = "t3.small", ami = "ami-456" }
    db  = { instance_type = "t3.medium", ami = "ami-789" }
  }
}

resource "aws_instance" "server" {
  for_each      = var.instances
  ami           = each.value.ami
  instance_type = each.value.instance_type

  tags = {
    Name = each.key
  }
}

# Access: aws_instance.server["web"], aws_instance.server["app"], etc.

Dynamic Blocks

Dynamic blocks generate repeated nested blocks within resources.

Dynamic Security Group Rules 
variable "ingress_rules" {
  type = list(object({
    port        = number
    protocol    = string
    cidr_blocks = list(string)
    description = string
  }))
  default = [
    { port = 80,  protocol = "tcp", cidr_blocks = ["0.0.0.0/0"], description = "HTTP" },
    { port = 443, protocol = "tcp", cidr_blocks = ["0.0.0.0/0"], description = "HTTPS" },
    { port = 22,  protocol = "tcp", cidr_blocks = ["10.0.0.0/8"], description = "SSH" }
  ]
}

resource "aws_security_group" "web" {
  name   = "web-sg"
  vpc_id = var.vpc_id

  dynamic "ingress" {
    for_each = var.ingress_rules
    content {
      from_port   = ingress.value.port
      to_port     = ingress.value.port
      protocol    = ingress.value.protocol
      cidr_blocks = ingress.value.cidr_blocks
      description = ingress.value.description
    }
  }

  egress {
    from_port   = 0
    to_port     = 0
    protocol    = "-1"
    cidr_blocks = ["0.0.0.0/0"]
  }
}

For Expressions

For Expression Examples 
# Transform list
variable "names" {
  default = ["web", "app", "db"]
}

locals {
  # List to list (uppercase)
  upper_names = [for name in var.names : upper(name)]
  # ["WEB", "APP", "DB"]

  # List to map
  name_map = { for name in var.names : name => "${name}-server" }
  # { web = "web-server", app = "app-server", db = "db-server" }

  # Filter list
  filtered = [for name in var.names : name if name != "db"]
  # ["web", "app"]

  # Transform map
  instance_arns = {
    for k, v in aws_instance.servers :
    k => v.arn
  }
}

# Output all instance IPs
output "instance_ips" {
  value = [for instance in aws_instance.web : instance.public_ip]
}

Conditional Examples

  count         = var.enable_nat_gateway ? 1 : 0

  value = var.enable_nat_gateway ? aws_eip.nat[0].public_ip : null

    "small"
  )
}

Template Files

templatefile() Usage 
# templates/user_data.sh.tpl
#!/bin/bash
echo "Environment: ${environment}"
echo "App Name: ${app_name}"
%{ for server in db_servers ~}
echo "DB Server: ${server}"
%{ endfor ~}

# main.tf
resource "aws_instance" "web" {
  ami           = var.ami_id
  instance_type = "t3.micro"

  user_data = templatefile("${path.module}/templates/user_data.sh.tpl", {
    environment = var.environment
    app_name    = "myapp"
    db_servers  = ["db1.example.com", "db2.example.com"]
  })
}

Interview Q&A

Q1: When should you use for_each over count?
  • Resources have meaningful identifiers (names)
  • Adding/removing items shouldn't affect others
  • Resources have different configurations
  • count: Identical resources, order doesn't matter
Q2: What are dynamic blocks?
  • Generate repeated nested blocks dynamically
  • Use for_each to iterate
  • Content block defines nested structure
  • Common for ingress/egress rules, tags
Q3: What is the try() function?
  • Evaluates expressions and returns first successful one
  • Handles errors gracefully
  • try(var.x.y.z, "default")
  • Useful for optional nested values
Q4: How do you create conditional resources?
  • Use count = condition ? 1 : 0
  • Or for_each = condition ? toset([1]) : []
  • Reference with [0] index for count
  • Check length before accessing
Q5: What is templatefile()?
  • Renders a template file with variables
  • Supports ${var} interpolation
  • Supports %{ for/if } directives
  • Returns string content
  • Common for user_data, configs
