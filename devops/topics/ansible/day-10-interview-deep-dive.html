🎯 Ansible Interview Q&A & Deep Dive

Comprehensive interview preparation covering Ansible fundamentals, advanced topics, troubleshooting, and real-world architecture questions.

Fundamentals

What is Ansible and how does it work?

Ansible is an agentless automation tool that connects to remote hosts over SSH (or WinRM for Windows) and executes tasks defined in YAML playbooks.

  • No agent required on managed nodes
  • Uses modules to perform actions
  • Idempotent: running playbooks multiple times yields the same result
What is an inventory file?

Inventory file lists hosts and groups for Ansible to manage. Can be static (INI/YAML) or dynamic (scripts/plugins).

  • Default: /etc/ansible/hosts
  • Custom: -i flag
What is a playbook?

Playbooks are YAML files describing automation tasks. They define plays (target hosts) and tasks (actions to perform).

Advanced Topics

How do you use variables and facts?
  • Variables: defined in playbooks, inventory, or external files
  • Facts: gathered automatically (ansible_facts)
  • Jinja2 templating: {{ variable }}
What are handlers and when are they used?

Handlers are special tasks triggered by notify when a change occurs. Used for actions like restarting services after config changes.

How do you secure secrets in Ansible?
  • Use Ansible Vault to encrypt files/variables
  • Never hardcode secrets in playbooks
  • Use --ask-vault-pass or vault password files

Troubleshooting

How do you debug a failing playbook?
  • Use debug module to print variables
  • Run with -v, -vvv for verbose output
  • Check SSH connectivity and permissions
  • Check for missing variables/files
What are common errors in Ansible?
  • YAML syntax errors (indentation, colons)
  • SSH connection failures
  • Module not found
  • Undefined variables

Architecture & Best Practices

How do you structure a large Ansible project?
  • Use roles for modularity and reuse
  • Group variables in group_vars and host_vars
  • Document roles and playbooks
  • Use Ansible Galaxy for community roles
How do you use Ansible in CI/CD?
  • Integrate ansible-playbook in pipeline steps
  • Lint playbooks with ansible-lint
  • Test roles with molecule
  • Use vault for secrets in CI

Quick Reference Commands

Essential Commands 
# Check version
ansible --version

# Run ad-hoc command
ansible all -m ping
ansible web -m shell -a 'uptime'

# Run playbook
ansible-playbook site.yml
ansible-playbook -i inventory.ini playbook.yml

# Lint playbooks
ansible-lint playbook.yml

# Encrypt/decrypt with vault
ansible-vault encrypt vars.yml
ansible-vault decrypt vars.yml
ansible-vault edit vars.yml

# Test roles
molecule test
