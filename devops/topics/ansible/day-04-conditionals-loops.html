🔄 Conditionals & Loops

Control flow in Ansible is handled with conditionals and loops, making playbooks flexible and powerful.

Conditionals (when)

  • Use when to run tasks only if a condition is true
  • Can use variables, facts, or expressions
Conditional Example 
- name: Install Apache on Ubuntu
  apt:
    name: apache2
    state: present
  when: ansible_facts['os_family'] == 'Debian'

Loops (with_items, loop)

  • Use with_items (legacy) or loop (modern) to repeat tasks
  • Register results for later use
Loop Example 
- name: Create multiple files
  file:
    path: "/tmp/{{ item }}"
    state: touch
  loop:
    - file1.txt
    - file2.txt

Debug & Error Handling

  • Use debug module to print variables
  • Use failed_when and ignore_errors for error handling
