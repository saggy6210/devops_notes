Kubernetes Interview Questions

1

What is Kubernetes?

Kubernetes is a portable, extensible, open-source platform for managing containerized workloads and services. It facilitates both declarative configuration and automation. Google open-sourced Kubernetes in 2014.

2

What are the key features Kubernetes provides?

  • Service discovery & load balancing: Expose containers via DNS or IP, distribute traffic
  • Storage orchestration: Auto-mount storage systems (local, cloud)
  • Automated rollouts & rollbacks: Change actual state to desired state
  • Automatic bin packing: Fit containers onto nodes based on resource needs
  • Self-healing: Restart failed containers, kill unresponsive ones
  • Secret & config management: Store and manage sensitive information
3

What are the components of Kubernetes Control Plane?

  • kube-apiserver: Exposes the Kubernetes API, front-end for control plane
  • etcd: Consistent, highly-available key-value store for all cluster data
  • kube-scheduler: Watches for new Pods, selects a node for them to run
  • kube-controller-manager: Runs controller processes (Node, Replication, Endpoints, Service Account)
4

What are the Node Components?

  • kubelet: Agent on each node, ensures containers run in Pods
  • kube-proxy: Network proxy maintaining network rules for Pod communication
  • Container Runtime: Software for running containers (Docker, containerd, CRI-O)
5

What are the default Kubernetes namespaces?

  • default: Default namespace for objects with no other namespace
  • kube-system: Namespace for objects created by the Kubernetes system
  • kube-public: Readable by all users, reserved for cluster usage
kubectl get namespaces
6

What is a Kubernetes Object?

Kubernetes Objects are persistent entities that represent the state of your cluster:

  • What containerized applications are running (and on which nodes)
  • Resources available to those applications
  • Policies around application behavior (restart, upgrades, fault-tolerance)
7

What fields are required in a Kubernetes YAML manifest?

  • apiVersion: Kubernetes API version to use
  • kind: Type of object to create (Pod, Deployment, Service)
  • metadata: Unique identification data (name, UID, namespace)
  • spec: Desired state of the object
8

How does Kubernetes DNS work?

When you create a Service, it creates a DNS entry in the format:

<service-name>.<namespace>.svc.cluster.local

Containers can resolve services using just <service-name> for same-namespace services.

9

What is a Pod?

A Pod is the smallest deployable unit in Kubernetes. It represents one or more containers that share storage, network, and a specification for how to run them.

10

What is a Deployment?

A Deployment provides declarative updates for Pods and ReplicaSets. You describe the desired state, and the Deployment Controller changes the actual state to match.

kubectl create deployment nginx --image=nginx
11

What is a Service in Kubernetes?

A Service is an abstraction that defines a logical set of Pods and a policy to access them. Service types:

  • ClusterIP: Internal cluster IP (default)
  • NodePort: Exposes on each node's IP at a static port
  • LoadBalancer: Exposes externally using cloud provider's LB
12

How do you get pods in a specific namespace?

kubectl --namespace=<namespace-name> get pods
kubectl get pods -n <namespace-name>
13

What is a ConfigMap?

A ConfigMap is an API object used to store non-confidential configuration data in key-value pairs. Pods can consume ConfigMaps as environment variables, command-line arguments, or config files in a volume.

14

What is a Secret?

A Secret is an object that contains sensitive data such as passwords, OAuth tokens, and SSH keys. Storing sensitive info in Secrets is safer than putting it in Pod definitions or container images.

15

What is an Ingress?

Ingress manages external access to services in a cluster, typically HTTP. It provides load balancing, SSL termination, and name-based virtual hosting.

16

What is a StatefulSet?

StatefulSet manages stateful applications. Unlike Deployments, it provides guarantees about ordering and uniqueness of Pods. Used for applications that require stable network identifiers, persistent storage, or ordered deployment.

17

What is a DaemonSet?

A DaemonSet ensures that all (or some) nodes run a copy of a Pod. As nodes are added, Pods are added to them. Use cases: log collection, monitoring agents, cluster storage daemons.

18

How to get cluster credentials for AKS?

az aks get-credentials --resource-group myRG --name myAKS
19

How to initialize a Kubernetes cluster with kubeadm?

kubeadm init --pod-network-cidr=10.244.0.0/16

After initialization, copy the kubeconfig:

mkdir -p $HOME/.kube
sudo cp /etc/kubernetes/admin.conf $HOME/.kube/config
20

What are common kubectl commands?

kubectl get nodes # List all nodes
kubectl get pods # List all pods
kubectl get pods -A # All namespaces
kubectl describe pod <name> # Pod details
kubectl logs <pod-name> # View logs
kubectl exec -it <pod> -- bash # Shell access
