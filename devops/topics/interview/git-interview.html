Git Interview Questions

1

What is Git?

Git is an open-source distributed version control system. It tracks changes in source code during software development and enables multiple developers to work on the same codebase.

2

How to configure Git with username and email?

git config --global user.name "Your Name"
git config --global user.email "your@email.com"
3

What is the difference between git init and git clone?

  • git init: Creates a new empty Git repository in the current directory
  • git clone: Creates a copy of an existing repository from a URL
git init # New repo
git clone [url] # Copy existing
4

What is the staging area in Git?

The staging area (index) is an intermediate area where commits are prepared. Files are added to staging with git add before committing.

git add file.txt # Stage single file
git add . # Stage all changes
5

What is the difference between git commit and git commit -a?

  • git commit -m "msg": Commits only staged changes
  • git commit -a -m "msg": Stages all tracked files and commits (skips untracked)
6

What does git diff do?

git diff shows file differences that are not yet staged.

git diff # Unstaged changes
git diff --staged # Staged changes
git diff branch1 branch2 # Between branches
7

What is the difference between git reset and git reset --hard?

  • git reset [commit]: Undoes commits, preserves changes locally (soft reset)
  • git reset --hard [commit]: Discards ALL history and changes, goes back to specified commit

Warning: --hard is destructive!

8

How to view commit history?

git log # Full log
git log --oneline # Compact view
git log --follow [file] # History of a file
9

How to work with branches?

git branch # List branches
git branch feature # Create branch
git checkout feature # Switch to branch
git checkout -b feature # Create and switch
git branch -d feature # Delete branch
10

What is the difference between git merge and git rebase?

  • git merge: Combines branches, creates a merge commit, preserves history
  • git rebase: Moves commits to new base, creates linear history, rewrites commits
git merge feature # Merge feature into current
git rebase main # Rebase current onto main
11

What is the difference between git pull and git fetch?

  • git fetch: Downloads changes but doesn't merge them
  • git pull: Downloads changes AND merges them (fetch + merge)
12

How to push changes to remote?

git push # Push to tracked remote
git push origin main # Push to specific remote/branch
git push --all origin # Push all branches
git push origin :branch-name # Delete remote branch
13

What is git stash?

git stash temporarily saves uncommitted changes so you can switch branches without committing.

git stash # Save changes
git stash list # List stashes
git stash pop # Apply and remove
git stash apply # Apply but keep
14

How to undo the last commit?

git reset --soft HEAD~1 # Keep changes staged
git reset HEAD~1 # Keep changes unstaged
git reset --hard HEAD~1 # Discard changes
15

How to disable SSL verification in Git?

git config --global http.sslVerify false

Only use this in development/testing environments!
