HTTP Status Codes

2xx Success

200 OK

Request succeeded. Response depends on HTTP method used.

4xx Client Errors

400 Bad Request

Server cannot understand request due to invalid syntax.

401 Unauthorized

Client must authenticate itself to get the requested response.

403 Forbidden

Client does not have access rights to the content.

404 Not Found

Server cannot find requested resource. URL not recognized.

408 Request Timeout

Server wants to shut down this unused idle connection.

5xx Server Errors

500 Internal Server Error

Server encountered an unexpected situation.

501 Not Implemented

Request method not supported by server.

502 Bad Gateway

Server as gateway got invalid response from upstream server.

503 Service Unavailable

Server not ready. Usually maintenance or overload.

504 Gateway Timeout

Server as gateway cannot get response in time.

Networking Interview Questions

1

What is DNS?

Domain Name System translates human-readable domain names (google.com) to IP addresses. It's like a phonebook for the internet.

2

What is a Load Balancer?

A load balancer distributes network traffic across multiple servers to ensure no single server becomes overwhelmed. Types:

  • L4 (Transport): Routes based on IP and port
  • L7 (Application): Routes based on content (URL, headers)
3

What is a Reverse Proxy?

A reverse proxy sits between clients and servers, forwarding client requests to appropriate backend servers. Benefits:

  • Load balancing
  • SSL termination
  • Caching
  • Security (hide backend servers)

Examples: Nginx, HAProxy, Traefik

4

What is the difference between TCP and UDP?

  • TCP: Connection-oriented, reliable, ordered delivery, error checking (HTTP, SSH, FTP)
  • UDP: Connectionless, faster, no guaranteed delivery (DNS, streaming, gaming)
5

What are common network ports?

  • 22: SSH
  • 80: HTTP
  • 443: HTTPS
  • 3306: MySQL
  • 5432: PostgreSQL
  • 6379: Redis
  • 27017: MongoDB
6

What is CIDR notation?

CIDR (Classless Inter-Domain Routing) is a method for allocating IP addresses. Format: IP/prefix

  • 10.0.0.0/8 - 16 million addresses
  • 192.168.1.0/24 - 256 addresses
  • 10.0.0.1/32 - Single host
7

What is SSL/TLS?

SSL (Secure Sockets Layer) and TLS (Transport Layer Security) are cryptographic protocols for secure communication. TLS is the successor to SSL.

Used for HTTPS, secure email, and other encrypted connections.

8

How to troubleshoot network connectivity?

ping host # Test connectivity
traceroute host # Trace path to host
nslookup domain # DNS lookup
dig domain # Detailed DNS query
curl -v url # HTTP request details
telnet host port # Test port connectivity
netstat -tulpn # List listening ports
9

What is a VPN?

Virtual Private Network creates an encrypted tunnel between your device and a VPN server, hiding your IP and securing your traffic. Used for remote access and security.

10

What is the difference between HTTP and HTTPS?

  • HTTP: Unencrypted, port 80, data sent in plaintext
  • HTTPS: Encrypted with SSL/TLS, port 443, secure data transfer
