Docker Interview Questions

1

What is Docker?

Docker is an open-source lightweight containerization technology. It allows you to package applications and their dependencies into standardized units called containers that can run consistently across different environments.

2

What is Docker Swarm?

Docker Swarm is a native clustering and orchestration tool for Docker. It helps you group multiple Docker hosts into a single virtual Docker host, enabling container orchestration, load balancing, and scaling.

3

What is Docker Hub?

Docker Hub is a cloud-based registry service that allows you to store and distribute Docker images. It also helps you link to code repositories and automate builds.

4

What is a Hypervisor?

A hypervisor allows you to create a virtual environment in which guest virtual machines operate. It controls the guest systems and manages resource allocation to the guests as necessary. Examples: VMware, Hyper-V, VirtualBox.

5

What is CNM (Container Networking Model)?

CNM stands for Container Networking Model. It's Docker's approach to networking that provides a pluggable architecture. Key components: Sandbox (network namespace), Endpoint (connects sandbox to network), Network (group of endpoints).

6

What is the default Docker network driver?

Docker provides different network drivers: bridge (default), host, overlay, and macvlan. Bridge is the default driver for standalone containers.

docker network create my-network
7

Which network driver should be used with Docker Swarm?

Use the overlay network driver with Docker Swarm. It enables the Swarm service by connecting multiple Docker host daemons together, allowing containers across different hosts to communicate.

8

What is the difference between CMD and ENTRYPOINT?

  • CMD: Sets default command and/or parameters, which can be overwritten from command line when container runs.
  • ENTRYPOINT: Configures a container that will run as an executable. Arguments are appended, not replaced.
9

What is the difference between ADD and COPY instruction?

  • COPY: Only copies local files or directories from the host into the Docker image.
  • ADD: Can use a URL as source and can extract tar files directly into the destination. URLs are downloaded but not decompressed.

Best practice: Use COPY unless you need ADD's specific features.

10

What is Docker Compose?

Docker Compose is a tool for defining and running multi-container Docker applications. You use a YAML file to configure your application's services, then create and start all services with a single command:

docker-compose up -d
11

What are docker save and docker load commands?

  • docker save: Saves one or more images to a tar archive. Contains all parent layers, tags, and versions.
  • docker load: Loads an image from a tar archive. Restores both images and tags.
docker save myimage > myimage.tar
docker load < myimage.tar
12

What are Docker container's possible states?

  • created: Container created but not started
  • restarting: Container being restarted
  • running: Currently running container
  • paused: Processes paused
  • exited: Container ran and completed (stopped)
  • dead: Daemon tried and failed to stop the container
13

What is the difference between RUN and CMD?

  • RUN: Executes commands at build time inside the Docker image. Commands are written as a new layer.
  • CMD: Defines default command to run when container starts. Only one CMD per Dockerfile.
14

What is the difference between Dockerfile and Docker image?

  • Dockerfile: A text file with instructions (recipe) for building Docker images.
  • Docker Image: The result produced when you build a Dockerfile. A read-only template.
  • Docker Container: A running instance of a Docker image.
15

How do you get the IP address of a Docker container?

docker inspect --format='{{.NetworkSettings.IPAddress}}' container_name
16

How do you connect/disconnect a container to a network?

docker network connect my_network container_name
docker network disconnect my_network container_name
