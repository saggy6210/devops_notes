CI/CD & Jenkins Interview Questions

1

What is CI/CD?

  • CI (Continuous Integration): Developers frequently merge code changes into a shared repository, with automated builds and tests
  • CD (Continuous Delivery): Code changes are automatically prepared for release to production
  • CD (Continuous Deployment): Every change that passes tests is automatically deployed to production
2

What is Jenkins?

Jenkins is an open-source automation server written in Java. It helps automate building, testing, and deploying software projects. It supports hundreds of plugins for integration with various tools.

3

How to install and start Jenkins?

java -jar jenkins.war --httpPort=8080

Access at http://localhost:8080

Initial password location:

/var/lib/jenkins/secrets/initialAdminPassword
4

What is a Jenkins Pipeline?

A Jenkins Pipeline is a suite of plugins that supports implementing and integrating continuous delivery pipelines. It's defined in a Jenkinsfile and can be Declarative or Scripted.

5

What is a Jenkinsfile?

A text file containing the definition of a Jenkins Pipeline. Checked into source control for Pipeline-as-Code.

pipeline {
agent any
stages {
stage('Build') {
steps { sh 'make' }
}
stage('Test') {
steps { sh 'make test' }
}
}
}
6

How to use parameters in Jenkins Pipeline?

parameters {
choice(name: 'env', choices: 'dev\ntest\nprod', description: 'Environment')
string(defaultValue: 'cli_creds', name: 'creds_id', description: 'Credentials')
}
7

How to use credentials in Jenkins Pipeline?

withCredentials([usernamePassword(
credentialsId: 'my-creds',
usernameVariable: 'USERNAME',
passwordVariable: 'PASSWORD')]) {
sh 'login --user=$USERNAME --pass=$PASSWORD'
}
8

What is a GitLab Runner?

GitLab Runner is an application that works with GitLab CI/CD to run jobs in a pipeline. It can be installed on various platforms and supports multiple executors (shell, docker, kubernetes).

gitlab-runner register
gitlab-runner verify
gitlab-runner list
9

What is a .gitlab-ci.yml file?

A YAML file that defines the GitLab CI/CD pipeline structure:

stages:
- build
- test
- deploy

build:
stage: build
script: echo "Building..."
tags:
- docker
10

What is SonarQube?

SonarQube is an open-source platform for continuous code quality inspection. It performs static code analysis to detect bugs, code smells, and security vulnerabilities.

11

How to fix SonarQube webhook issues in Jenkins?

If SonarQube is in pending state when triggered:

  1. Go to SonarQube Administration → Webhooks
  2. Update with correct Jenkins webhook URL
  3. Save and verify connection
12

How to secure Jenkins?

  • Go to Jenkins → Global Security Configuration
  • Enable Matrix-based security
  • Remove Anonymous user access
  • Configure proper user permissions
13

What are deployment strategies?

  • Rolling: Gradually replace instances
  • Blue-Green: Two identical environments, switch traffic
  • Canary: Deploy to subset of users first
  • Recreate: Stop old version, start new
14

What is a webhook?

A webhook is an HTTP callback that sends data to a URL when an event occurs. Used to trigger CI/CD pipelines automatically when code is pushed.

Example GitLab webhook with credentials:

https://user:pass@jenkins.example.com/github-webhook/
15

What are Jenkins agents/nodes?

Jenkins agents (formerly slaves) are machines that execute builds dispatched by the master. They help distribute workload and run builds in different environments.
